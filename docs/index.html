Methodology

How Prediction Works

The system processes clinical inputs through a validated machine learning pipeline to produce calibrated risk probabilities.

01
Data Input
Patient enters 8 clinical health parameters including glucose, blood pressure, BMI, and insulin levels.
02
Preprocessing
Values are validated, standardized using StandardScaler, and prepared for model inference.
03
ML Classification
The best-performing model (auto-selected via cross-validation) computes prediction probabilities.
04
Risk Assessment
Probability is stratified into risk categories with clinical insights and personalized recommendations.

Support

Frequently Asked Questions

How accurate is this prediction model?
The model is trained on the Pima Indians Diabetes Dataset and evaluated using 5-fold stratified cross-validation. Three algorithms are compared and the best-performing one is automatically selected. This is a screening tool and should not be used as a sole diagnostic indicator.
What dataset is used for training?
The Pima Indians Diabetes Dataset from the National Institute of Diabetes and Digestive and Kidney Diseases. It contains 768 patient records with 8 clinical features and a binary outcome variable.
Is my health data stored or shared?
No. On this deployment, all predictions run entirely in your browser using JavaScript. No data is sent to any server, stored, or shared with any third party.
Can this replace a medical diagnosis?
No. This tool is built for educational and screening purposes only. It does not constitute medical advice. Always consult a qualified healthcare professional for diagnosis, treatment, and medical decisions.
What machine learning algorithms are used?
Three algorithms are evaluated: Logistic Regression, Random Forest (200 estimators, max depth 8), and Gradient Boosting (150 estimators, learning rate 0.1). Each is wrapped in a scikit-learn Pipeline with StandardScaler preprocessing. The best model is selected automatically via cross-validation accuracy.
